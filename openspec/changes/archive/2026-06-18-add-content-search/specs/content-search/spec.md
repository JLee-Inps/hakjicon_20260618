## ADDED Requirements

### Requirement: 키워드로 도서 검색

시스템은 사용자가 입력한 키워드로 도서 콘텐츠의 제목(`title`)과 설명(`description`)을 부분일치(대소문자 무시)로 검색하여, 일치하는 도서 목록을 반환해야 한다(SHALL).

#### Scenario: 일치하는 도서가 있는 검색

- **WHEN** 사용자가 검색창에 "상담"을 입력하고 검색을 실행한다
- **THEN** 시스템은 `title` 또는 `description`에 "상담"을 포함하는 모든 도서를 결과 카드 목록으로 보여준다

#### Scenario: 대소문자/부분 문자열 일치

- **WHEN** 사용자가 도서 제목의 일부 문자열만 입력한다
- **THEN** 시스템은 해당 문자열을 포함하는 도서를 대소문자 구분 없이 결과에 포함한다

### Requirement: 결과 없음 처리

시스템은 검색 결과가 없을 때 빈 화면이 아니라 "결과 없음"을 알리는 안내 메시지를 보여줘야 한다(SHALL).

#### Scenario: 일치하는 도서가 없는 검색

- **WHEN** 사용자가 어떤 도서와도 일치하지 않는 키워드를 입력하고 검색한다
- **THEN** 시스템은 결과 카드 대신 "검색 결과가 없습니다" 안내 메시지를 표시한다

### Requirement: 빈 검색어 처리

시스템은 검색어가 비어 있을 때 검색을 수행하지 않거나 전체/안내 상태를 유지해야 하며(SHALL), 오류를 발생시키지 않아야 한다(SHALL).

#### Scenario: 빈 검색어로 검색 시도

- **WHEN** 사용자가 검색어를 입력하지 않고 검색을 실행한다
- **THEN** 시스템은 오류 없이 초기 안내 상태를 유지하고 검색 쿼리를 보내지 않는다

### Requirement: 검색 결과 카드 표시

시스템은 각 검색 결과를 카드 형태로 보여주며, 카드에는 최소한 제목과 저자, 분류(category)를 포함해야 한다(SHALL).

#### Scenario: 결과 카드 내용

- **WHEN** 검색 결과 도서가 화면에 렌더링된다
- **THEN** 각 카드는 해당 도서의 제목, 저자, 분류를 표시한다
