## ADDED Requirements

### Requirement: 키워드로 도서 검색

시스템은 사용자가 입력한 키워드로 도서의 제목(`title`)과 설명(`description`)을 부분일치(대소문자 무시)로 검색하여 일치하는 도서 목록을 반환해야 한다(SHALL). 빈 검색어에는 쿼리를 보내지 않아야 한다(SHALL).

#### Scenario: 일치하는 도서가 있는 검색

- **WHEN** 사용자가 검색창에 "상담"을 입력하고 검색을 실행한다
- **THEN** 시스템은 `title` 또는 `description`에 "상담"을 포함하는 도서를 결과 목록으로 보여준다

#### Scenario: 결과 없음

- **WHEN** 사용자가 어떤 도서와도 일치하지 않는 키워드로 검색한다
- **THEN** 시스템은 "검색 결과가 없습니다" 안내를 표시한다

#### Scenario: 빈 검색어

- **WHEN** 사용자가 검색어 없이 검색을 실행한다
- **THEN** 시스템은 쿼리를 보내지 않고 초기 안내 상태를 유지한다

### Requirement: 인기 추천

시스템은 홈 화면에서 인기 점수(`view_count`) 기준 상위 N개(기본 6개) 도서를 추천 목록으로 보여줘야 한다(SHALL). 추천 목록의 각 항목은 제목과 저자, 분류를 포함해야 한다(SHALL).

#### Scenario: 홈 진입 시 인기 추천 노출

- **WHEN** 사용자가 검색 없이 홈 화면에 진입한다
- **THEN** 시스템은 인기 점수 내림차순 상위 6개 도서를 "인기 추천"으로 표시한다

#### Scenario: 도서 조회로 인기 점수 반영

- **WHEN** 사용자가 한 도서의 상세를 조회한다
- **THEN** 시스템은 해당 도서의 `view_count`를 1 증가시켜 이후 인기 순위에 반영한다

### Requirement: 북마크 저장

시스템은 사용자가 특정 도서를 북마크로 저장할 수 있게 해야 하며(SHALL), 인증 없이 브라우저별(localStorage client id) 익명 저장을 지원해야 한다(SHALL). 이미 북마크된 도서는 다시 저장되지 않아야 한다(SHALL).

#### Scenario: 도서 북마크 저장

- **WHEN** 사용자가 도서 카드의 북마크 버튼을 누른다
- **THEN** 시스템은 해당 도서를 현재 브라우저의 client id로 `bookmarks`에 저장하고 버튼을 "저장됨" 상태로 표시한다

#### Scenario: 중복 북마크 방지

- **WHEN** 사용자가 이미 북마크한 도서의 북마크 버튼을 다시 누른다
- **THEN** 시스템은 중복 행을 만들지 않고 저장 상태를 유지한다
